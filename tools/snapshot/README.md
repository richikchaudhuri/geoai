# geoai-snapshot

A Rust CLI that freezes the live Supabase `photos` + `assessments` tables
into a single static file. Two roles:

1. **Offline demo fallback** for the WebGIS. When campus Wi-Fi or
   Supabase is unreachable, the WebGIS can load the snapshot file instead
   of querying the live database.
2. **Citable artifact** for the research report â€” a snapshot of the
   dataset at a specific point in time, hashable, reproducible, embeddable.

## Build

```powershell
cd tools/snapshot
cargo build --release
# binary: ./target/release/geoai-snapshot.exe
```

Rust 1.75+ required.

## Quick start

```powershell
# Defaults: format=json, output=data/snapshot.json, minified.
.\target\release\geoai-snapshot.exe

# Pretty-printed (easier to diff in a PR review)
.\target\release\geoai-snapshot.exe --pretty

# GeoJSON for QGIS / kepler.gl / ArcGIS
.\target\release\geoai-snapshot.exe --format geojson

# Both formats side-by-side
.\target\release\geoai-snapshot.exe --both --pretty
```

### Flags

| Flag | Default | Purpose |
|---|---|---|
| `--output` | `data/snapshot.{json\|geojson}` | Output path. |
| `--format` | `json` | `json` (WebGIS shape) or `geojson` (FeatureCollection). |
| `--both` | off | Write both formats side-by-side, swapping extension. |
| `--pretty` | off | Indent the JSON. Larger file, easier diff. |
| `--limit` | 10000 | Row cap per table (Supabase REST max). |

## Output formats

### `--format json` â€” WebGIS-compatible shape

Matches what `app.js` already fetches from Supabase, so the WebGIS can
load it as a drop-in replacement.

```json
{
  "photos":      [{...}, {...}, ...],
  "assessments": [{...}, {...}, ...],
  "fetchedAt":   "2026-05-19T11:23:45Z",
  "_meta": {
    "tool": "geoai-snapshot/0.1.0",
    "source": "https://vtlkitpoffudiefuoijb.supabase.co",
    "photo_count": 247,
    "assessment_count": 198
  }
}
```

### `--format geojson` â€” portable GeoJSON FeatureCollection

Each feature is a photo merged with its matching assessment. Opens in
QGIS, ArcGIS, kepler.gl, geojson.io, mapshaper, etc.

```json
{
  "type": "FeatureCollection",
  "metadata": {
    "generated_at": "2026-05-19T11:23:45Z",
    "source": "https://vtlkitpoffudiefuoijb.supabase.co",
    "tool": "geoai-snapshot/0.1.0",
    "photo_count": 247,
    "assessment_count": 198,
    "feature_count": 247,
    "classified_count": 198
  },
  "features": [
    {
      "type": "Feature",
      "geometry": { "type": "Point", "coordinates": [77.5946, 12.9716] },
      "properties": {
        "photo_id": "...",
        "assessment_id": "...",
        "address": "MG Road, Bengaluru, Karnataka 560001, India",
        "image_url": "https://res.cloudinary.com/.../abc.jpg",
        "status": "classified",
        "distress_types": ["pothole"],
        "severity": "Medium",
        "stage1_confidence": 0.94,
        "stage2_confidence": 0.87,
        "created_at": "...",
        "processed_at": "...",
        "expert_reviewed": false
      }
    },
    ...
  ]
}
```

**GeoJSON coords are `[lng, lat]`** (longitude first) per RFC 7946. Lookups
work with QGIS, ArcGIS, Leaflet (`L.geoJSON()`), Mapbox, etc.

## Wiring it into the WebGIS as a fallback

To make the WebGIS fall back to the snapshot when Supabase is unreachable,
add this near the start of `loadAssessments()` in `app.js` (sketch):

```js
if (sb) {
  try {
    // ... existing direct-Supabase code ...
  } catch (e) {
    console.warn('[GeoAI] Supabase failed, loading snapshot fallback', e);
    const snap = await fetch('data/snapshot.json').then(r => r.json());
    photos = snap.photos || [];
    assessments = snap.assessments || [];
  }
}
```

Commit the snapshot to the repo (`git add data/snapshot.json`) and the
GitHub Pages deploy will serve it as a static asset alongside the site.

## Dependencies

| Crate | Purpose |
|---|---|
| `tokio` | async runtime + `tokio::join!` for parallel table fetches |
| `reqwest` (rustls-tls) | HTTPS client, no OpenSSL |
| `clap` | CLI parsing with derive |
| `serde_json` | JSON serialization |
| `anyhow` | error handling |

No `chrono`, no `time` â€” ISO 8601 timestamps are generated by hand using
Howard Hinnant's civil-from-days algorithm.

## Notes

- The Supabase REST cap is 10000 rows per request. If the dataset ever
  exceeds that, this tool will need pagination via the `Range` header.
- Anon JWT is baked in â€” same key the WebGIS ships, RLS allows public
  reads on these tables.
- Output is minified by default to keep the snapshot small for GitHub
  Pages. Pass `--pretty` for human-diffable output (e.g., in PRs).
